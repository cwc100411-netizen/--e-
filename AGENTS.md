# Project Rules for Codex

本项目是 STM32F103C8T6 工程，使用 Keil5 编译。

## 基本要求

1. 默认使用中文回答。
2. 不要大规模重构，除非我明确要求。
3. 优先做最小、最有效的修改。
4. 修改代码前，先说明你准备改哪些文件、为什么改。
5. 修改代码后，必须列出与原来版本的差异点。
6. 修改代码后，必须列出：
   - 修改了哪些文件
   - 修改了哪些函数
   - 修改目的
   - 可能影响的地方
   - 如何在 Keil5 中验证
7. 不要随意新增模块。
8. 不要随意改动硬件引脚定义。
9. 如果发现需求不清楚，先给出你的理解和假设，不要直接乱改。


## 完成标准

完成任务前必须：

1. 给出修改总结。
2. 给出代码差异点。
3. 给出 Keil5 编译检查建议。
4. 给出实际测试步骤。
5. 明确说明是否改动了原有逻辑。
