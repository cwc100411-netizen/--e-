# AGENTS.md

## 项目说明

本项目基于 **STM32F103C8T6**，用于控制二维云台、两个 42 步进电机、双路步进电机驱动模块、激光模块和 Maxicam 摄像机。

硬件清单：

* STM32F103C8T6
* 两个 42 步进电机
* 双路步进电机驱动模块
* 激光模块
* 二维云台
* Maxicam 摄像机

目标：通过 STM32 控制两个步进电机，使二维云台完成水平和垂直方向运动，并结合摄像机数据实现目标定位或激光指向。

## 当前步进电机引脚定义

步进电机驱动采用 `STEP / DIR / EN` 三线控制方式，其中 `STEP` 由定时器 PWM 输出。

* 竖直方向电机：电机1
  * `PA1`：`STEP`，对应 `TIM2_CH2`
  * `PB4`：`DIR`
  * `PB8`：`EN`
* 水平方向电机：电机2
  * `PA7`：`STEP`，对应 `TIM3_CH2`
  * `PB1`：`DIR`
  * `PB3`：`EN`

调用 `Stepper_TurnAngleBoth(90, 90);` 时，第一个参数控制水平方向电机，第二个参数控制竖直方向电机。


## Codex 工作规则

你是本项目的嵌入式代码助手，请遵守以下规则：

1. 所有说明使用中文。
2. 优先写简单、稳定、适合新手理解的代码。
3. 不要随意大规模重构项目。
4. 不要擅自修改芯片型号、引脚定义、定时器配置。
5. 修改代码前，必须先阅读相关文件。
6. 每次修改代码后，必须列出与原版本的差异点。
7. 重要代码必须添加中文注释。
8. 不确定硬件连接或通信协议时，不要凭空编写代码。
9. 不要使用复杂宏、函数指针、动态内存等不适合新手调试的写法。
10. `main.c` 只负责主流程，底层驱动尽量放到独立模块中。

## main.c 编写要求

`main.c` 应保持简洁，只负责初始化和主循环。

推荐结构：

```c
int main(void)
{
    System_Init();

    Stepper_Init();
    Gimbal_Init();
    Laser_Init();
    Camera_Init();

    Laser_OFF();

    while (1)
    {
        App_Run();
    }
}
```

不要把步进电机底层控制、摄像机协议解析、激光逻辑全部堆在 `main.c` 中。
